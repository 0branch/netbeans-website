<div class="sect2">
<h3 id="_how_do_i_create_a_patch_for_a_preexisting_netbeans_module">How do I create a patch for a preexisting NetBeans module?</h3>
<div class="paragraph">
<p>If you need to patch an existing module, you can place a JAR file relative to the original.
For example, to patch <code>ide/modules/org-openide-example.jar</code>
you make a JAR like <code>ide/modules/patches/org-openide-example/mypatch.jar</code>.
The <code>mypatch</code> part of your JAR file patch can be named anything you like.
The JAR file should only contain those classes you want to patch.
It does not need a manifest, though an empty manifest is harmless.</p>
</div>
<div class="paragraph">
<p>The patch must be in the same cluster as the original. ()
If you want to create an NBM containing a patch,
you must ensure it will be installed in the same cluster
(use the <code>nbm.target.cluster</code> property),
but note that you cannot test such a dummy module as part of a module suite
(since this property is interpreted only by Plugin Manager).
If you are distributing a complete application including a patch to the NB Platform,
you will need to either manually preinstall the patch JAR in your copy of the Platform;
or override your <code>build-zip</code> target to include the JAR in the final ZIP
(in which case testing using <strong>Run Project</strong> will not have the patch active).</p>
</div>
<div class="paragraph">
<p>&lt;hr/&gt;
Applies to: NetBeans 6.x</p>
</div>
</div>
