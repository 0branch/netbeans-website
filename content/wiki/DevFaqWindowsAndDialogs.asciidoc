<div class="sect1">
<h2 id="_windows_dialogs">Windows &amp; dialogs</h2>
<div class="sectionbody">
<div class="sect2">
<h3 id="_can_i_just_open_up_my_own_frames_and_dialogs">Can I just open up my own frames and dialogs?</h3>
<div class="ulist">
<ul>
<li>
<p>Yes, you could; but please instead use the standard windowing system to do these sorts of things. Then you will properly handle docking, separated floating windows, context help, various keyboard shortcuts, and many other things contributing to the IDE&#8217;s appearance and functionality.</p>
</li>
<li>
<p>The Window System API describes the general steps you should use to open up new function windows, if you need such a thing. Specifically, you should use `TopComponent`s for such purposes.</p>
</li>
<li>
<p>You can also use <code>DialogDisplayer.notify(NotifyDescriptor)</code> to show dialogs that interact well with the IDE&#8217;s window system, and have a number of bits of prebuilt UI. You can use various standard subclasses of <code>NotifyDescriptor</code> to represent simple messages; exceptions; general-purpose dialogs with content panels; or even multi-stage wizards.</p>
</li>
</ul>
</div>
</div>
<div class="sect2">
<h3 id="_what_are_the_steps_to_create_a_simple_wizard">What are the steps to create a simple Wizard?</h3>
<div class="paragraph">
<p><em>Easiest way is to use File &#8594; New File wizard, Module Development category and Wizard item, which will generate all needed boilerplate
code for you. Essentially what wizard does is described below:</em></p>
</div>
<div class="sect3">
<h4 id="_create_panels">Create Panels:</h4>
<div class="paragraph">
<p>You should start with creating a set (at least one) on WizardDescriptor.Panel objects (see <a href="http://performance.netbeans.org/howto/dialogs/wizard-panels.html">Wizard Panel Performance Guide</a> for more information about the best way to create a panel).</p>
</div>
</div>
<div class="sect3">
<h4 id="_create_wizarddescriptor">Create WizardDescriptor</h4>
<div class="paragraph">
<p>Use the panels to tell the a WizardDescriptor instance the sequence of panels it should display.
This you can do either directly by <code>WizardDescriptor wd = WizardDescriptor(wizardPanelArray)</code>
Or you can create a <code>WizardDescriptor.Iterator</code> with these panels, which gives you more control over the sequencing.</p>
</div>
</div>
<div class="sect3">
<h4 id="_set_properties">Set Properties</h4>
<div class="paragraph">
<p>Set certain properties on the WizardDescriptor which can influence the appearence of the wizard. If you like to add a help pane for example you call:
<code>wd.putProperty("WizardPanel_autoWizardStyle",Boolean.TRUE);
wd.putProperty("WizardPanel_helpDisplayed",Boolean.TRUE);</code></p>
</div>
<div class="paragraph">
<p>This will display a help html file which has to be defined on each panel by setting a <code>clientProperty</code> in the JComponent superclass of the <strong>panel</strong> that is the wizard content. In this case it would look like:</p>
</div>
<div class="paragraph">
<p><code>putClientProperty("WizardPanel_helpURL",new URL("<a href="http://path/to/help/html/file/panelHelp.html">http://path/to/help/html/file/panelHelp.html</a>"));</code></p>
</div>
</div>
<div class="sect3">
<h4 id="_show_wizard">Show Wizard</h4>
<div class="paragraph">
<p>Finally you set the Wizard to screen using the <code>DialogDisplayer</code></p>
</div>
<div class="listingblock">
<div class="content">
<pre class="prettyprint highlight"><code class="language-java" data-lang="java">Dialog d = DialogDisplayer.getDefault().createDialog(wd);

        d.setVisible(true);

        d.toFront();</code></pre>
</div>
</div>
</div>
</div>
