<div class="sect1">
<h2 id="_which_way_should_i_use_to_register_an_object_in_the_default_lookup">Which Way Should I Use To Register an Object in the Default Lookup?</h2>
<div class="sectionbody">
<div class="paragraph">
<p>There are two ways to put something in the <a href="DevFaqLookupDefault.asciidoc">default Lookup</a>:</p>
</div>
<table class="tableblock frame-all grid-all spread">
<colgroup>
<col style="width: 33.3333%;">
<col style="width: 33.3333%;">
<col style="width: 33.3334%;">
</colgroup>
<thead>
<tr>
<th class="tableblock halign-left valign-top">What</th>
<th class="tableblock halign-left valign-top">When to Use It</th>
<th class="tableblock halign-left valign-top">How</th>
</tr>
</thead>
<tbody>
<tr>
<td class="tableblock halign-left valign-top"><p class="tableblock">Use the <code>@ServiceProvider</code> annotation</p></td>
<td class="tableblock halign-left valign-top"><p class="tableblock">Most of the time, unless you are building with Maven.  This annotation relies on NetBeans' build infrastructure to generate flat files in <code>META-INF/services</code> described below, at compile time.</p></td>
<td class="tableblock halign-left valign-top"><p class="tableblock">Annotate your file with <code>@ServiceProvider(service=Something.class)</code></p></td>
</tr>
<tr>
<td class="tableblock halign-left valign-top"><p class="tableblock">Put an <code>.instance</code> file in your module layer in <code>Services/Hidden</code></p></td>
<td class="tableblock halign-left valign-top"><p class="tableblock">Rarely - useful if you are making services available dynamically on the fly, for example when <a href="DevFaqDynamicSystemFilesystem.asciidoc">dynamically changing layer contents</a>, or when <a href="DevFaqFileAttributes.asciidoc">specialized registration metadata</a> or factory methods are needed</p></td>
<td class="tableblock halign-left valign-top"><p class="tableblock">Declare a <a href="DevFaqInstanceDataObject.asciidoc">.instance file</a>, a <a href="DevFaqDotSettingsFiles.asciidoc">.settings file</a> or similar, or a <a href="DevFaqDotShadowFiles.asciidoc">.shadow file</a> in your module&#8217;s <a href="DevFaqModulesLayerFile.asciidoc">layer.xml file</a></p></td>
</tr>
</tbody>
</table>
